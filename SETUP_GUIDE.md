# Claude Code セットアップガイド — 未経験者向け

このガイドは、まだ VSCode + Claude Code を使っていない人向けの導入手順です。所要時間は約30分です。

---

## このガイドの対象者

このガイドは、以下のいずれかに該当する人だけ読んでください：

- これから Claude Code を使ってみたい人
- すでに Claude Code 導入を検討中で、具体的な手順を知りたい人

すでに VSCode で Claude Code を使っている人は、このガイドは飛ばして `START_HERE.md` に進んでください。

**Claude Code を使わない人は、このガイドおよび ZIP 内の他のファイルは無視してかまいません。** ただし、開発しているアプリのセキュリティ確保は引き続き各自の責任で実施してください。

---

## 重要なルール：Claude Code は VSCode のターミナルから起動すること

ナイスネイルグループでは、Claude Code を使う場合は **必ず VSCode のターミナル（Terminal → New Terminal）から `claude` コマンドで起動する** ことをルールとしています。

Windows Terminal、PowerShell、コマンドプロンプト等の単独使用は禁止です。理由：

- VSCode が開いているフォルダと Claude Code の作業ディレクトリが自動で揃う
- Claude Code が編集したファイルを即座に VSCode で確認できる
- 環境差異によるトラブル発生を防ぐ
- 他のメンバーがサポートする際に画面構成が統一されている

このガイドの手順も、すべて VSCode のターミナル使用を前提に書かれています。

---

## なぜ Claude Code が推奨されるのか

このテンプレート一式（ZIPに含まれるファイル群）は、Claude Code 使用を前提に設計されています。Claude Code を使うと以下のメリットがあります：

- プロジェクト全体のセキュリティリスクを Claude が自動で点検
- 認証情報の漏洩リスク（API キー・パスワードのコード混入など）を発見
- 危険なコマンド（ファイル削除など）の事故防止
- 開発ルール・セキュリティ基準の自動適用

ブラウザ版の Claude（claude.ai）でもコードの相談はできますが、プロジェクト全体の自動点検やファイル配置の自動化はできません。本格的な開発をしている人ほど、Claude Code の導入価値が高くなります。

## 必要なもの

- Windows パソコン
- インターネット接続
- ナイスネイルのチームプランで使っている Claude のアカウント
- 30分程度の時間

---

## Step 1: Node.js をインストール（5分）

Claude Code を動かすために必要なソフトウェアです。

1. ブラウザで https://nodejs.org/ja を開く
2. 大きく表示されている「LTS」と書かれた緑のボタンをクリック（推奨版がダウンロードされる）
3. ダウンロードされた `.msi` ファイルをダブルクリック
4. インストーラーが起動するので「Next」を押し続け、最後に「Install」
5. 「ユーザーアカウント制御」のダイアログが出たら「はい」
6. 「Finish」を押して完了

### インストール確認

1. Windows キー + R を押して「ファイル名を指定して実行」を開く
2. `cmd` と入力して Enter（黒い画面が開く）
3. 以下を入力して Enter：
   ```
   node --version
   ```
4. `v20.xx.x` のようなバージョン番号が表示されればOK
5. 続けて以下も入力：
   ```
   npm --version
   ```
6. こちらもバージョン番号が表示されればOK

うまく動かない場合はパソコンを再起動してから再度試してください。

---

## Step 2: VSCode をインストール（5分）

すでに VSCode をインストール済みの人はこの Step は飛ばしてください。

1. ブラウザで https://code.visualstudio.com/ を開く
2. 「Download for Windows」をクリック
3. ダウンロードされた `.exe` ファイルをダブルクリック
4. ライセンス契約に同意して「次へ」
5. 「追加タスクの選択」で以下にチェック：
   - 「エクスプローラーのファイル コンテキスト メニューに『Code で開く』アクションを追加する」
   - 「エクスプローラーのディレクトリ コンテキスト メニューに『Code で開く』アクションを追加する」
6. 「次へ」「インストール」を押して完了

**Step 3 以降の作業は、すべて VSCode のターミナルから行います。** Windows のコマンドプロンプトや PowerShell は使いません。

---

## Step 3: Claude Code をインストール（5分）

VSCode のターミナルから Claude Code をインストールします。

1. VSCode を起動
2. 上部メニュー「Terminal」→「New Terminal」をクリック（または Ctrl + @）
3. 画面下にターミナルが開く
4. 以下を入力して Enter：
   ```
   npm install -g @anthropic-ai/claude-code
   ```
5. しばらく待つとインストールが完了（数分かかる）
6. 完了後、以下を入力して動作確認：
   ```
   claude --version
   ```
7. バージョン番号が表示されればOK

### インストール時にエラーが出る場合

権限エラー（EACCES など）が出たら、ターミナルを「管理者として実行」してから再度試してください。

---

## Step 4: Claude Code にログイン（3分）

1. VSCode のターミナルで以下を入力：
   ```
   claude
   ```
2. 初回起動時にログイン方法を聞かれる
3. 「Claude.ai account」を選択（チームプランのアカウント）
4. ブラウザが自動で開き、Claude のログイン画面が表示される
5. ナイスネイルのチームプランで使っているアカウントでログイン
6. 「Authorize」ボタンをクリック
7. ブラウザに「Login Successful」と表示されたらブラウザを閉じる
8. ターミナルに戻ると Claude Code が起動状態になっている

### ログイン確認

ターミナル内で何か質問してみてください。例：
```
こんにちは、動作確認です
```

Claude が返答すれば正常です。

---

## Step 5: VSCode の設定（任意・推奨）

### 日本語化（英語に慣れていない人向け）

1. VSCode 左側のアイコン群から、四角が積み重なったような「拡張機能」アイコンをクリック
2. 検索欄に `Japanese Language Pack` と入力
3. 「Japanese Language Pack for Visual Studio Code」をインストール
4. 右下に「Restart」と出るのでクリック → VSCode が日本語化される

### Claude Code 拡張機能（任意）

ターミナルだけでなく VSCode の機能としても Claude Code を使いたい場合：

1. 拡張機能アイコンをクリック
2. 検索欄に `Claude Code` と入力
3. Anthropic 公式の拡張機能をインストール

これは任意です。ターミナル版だけでも今回の作業はできます。

---

## Step 6: 動作確認（5分）

簡単なフォルダで Claude Code を起動してみましょう。

1. デスクトップに「test_claude」というフォルダを作成（右クリック→新規作成→フォルダー）
2. VSCode の「ファイル」→「フォルダーを開く」で `test_claude` を選択
3. ターミナルを開く（Terminal → New Terminal）
4. 以下を入力：
   ```
   claude
   ```
5. Claude Code が起動したら、以下を試す：
   ```
   このフォルダに hello.txt というファイルを作成して、中身は「動作確認」と書いてください
   ```
6. Claude Code が確認を求めてきたら承認
7. VSCode 左側のファイル一覧に `hello.txt` が作成されたらOK

---

## Step 7: 自分のプロジェクトに移行

動作確認が終わったら、本来のプロジェクトフォルダで作業を開始します。

1. VSCode で「ファイル」→「フォルダーを開く」
2. 自分が開発しているプロジェクトのフォルダを選択
3. ターミナルを開く
4. 以下のファイルをそのプロジェクトのフォルダにコピー（共有された ZIP の中身）：
   - `START_HERE.md`
   - `CLAUDE.md`
   - `CLAUDE_security_baseline.md`
   - `NETLIFY_DEPLOYMENT.md`
   - `.claude/settings.json`（`.claude` フォルダごと）
   - `.gitignore.template`
   - `security_audit_prompt.md`
5. `START_HERE.md` を VSCode で開く
6. その中に書かれている指示に従う

## トラブルシューティング

### `node --version` でバージョンが表示されない
パソコンを再起動してから再度試してください。それでもダメな場合は Node.js を再インストール。

### `claude` コマンドが「コマンドが見つかりません」と出る
1. ターミナルを一度閉じて再度開く
2. それでもダメな場合は VSCode を再起動
3. それでもダメな場合は Step 3 のインストールを再実行

### ログインのブラウザが開かない
ターミナルに表示される URL を手動でコピーしてブラウザに貼り付けてください。

### `npm install` で時間がかかりすぎる
社内ネットワークのプロキシが影響している可能性があります。池宮に確認してください。

### 何度もパスワードを聞かれる
Windows の権限関連です。コマンドプロンプトを「管理者として実行」して再度試してください。

### その他のエラー
エラーメッセージのスクリーンショットを撮って池宮に相談してください。

## よくある質問

### Q1: チームプランの料金は別途かかる？
いいえ。すでにナイスネイルのチームプランで Claude を使っているなら追加料金はかかりません。

### Q2: ブラウザの Claude（claude.ai）も引き続き使える？
使えます。今回 VSCode 版を追加で入れる形です。両方併用可能です。

### Q3: 既に開発中のプロジェクトに影響はある？
基本的にありません。VSCode + Claude Code はあくまで開発支援ツールであり、プロジェクト本体には影響しません。

### Q4: ローカルだけで開発していて GitHub を使っていない場合は？
今回の作業には GitHub は必須ではありません。VSCode + Claude Code だけで完結します。

### Q5: Mac でも同じ手順でできる？
基本的に同じですが、Node.js のインストール方法が少し異なります。Mac の場合は homebrew を使うのが推奨ですが、Windows のインストーラーと同じく公式サイトから `.pkg` でもインストールできます。

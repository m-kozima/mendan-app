# 開発環境セキュリティ点検＆ルール設置 — 実行手順

## このファイルの前提

**このファイルは VSCode のターミナルから Claude Code を起動して使用することが前提です。**

VSCode 内のターミナルで Claude Code を起動することをルールとしています。Windows Terminal や PowerShell 単独で起動するのは禁止です。理由：

- VSCode が開いているフォルダと Claude Code の作業ディレクトリが自動で揃う
- Claude Code が編集したファイルを即座に VSCode で確認できる
- 環境差異によるトラブル発生を防ぐ
- 他のメンバーがサポートする際に画面構成が統一されている

Claude Code をまだ使っていない人は、まず `SETUP_GUIDE.md` を見てセットアップしてからこのファイルに戻ってきてください。または、Claude Code を使わずに自己責任でセキュリティ確保するという判断でも構いません（ただし、本テンプレートの自動点検機能は使えません）。

---

## このファイルの目的

あなたの開発プロジェクトに対して、Claude Code が自動で以下を実行します：

1. セキュリティリスクの調査
2. 開発ルール md の適切な配置
3. .gitignore の更新
4. 改善提案

あなたがやることは2つだけです。

---

## やること（2ステップ）

### Step 1: テンプレート一式をプロジェクトのルートフォルダにコピー

以下のファイルを、点検したいプロジェクトのフォルダ直下にコピーしてください：

- `START_HERE.md`（このファイル）
- `CLAUDE.md`
- `CLAUDE_security_baseline.md`
- `NETLIFY_DEPLOYMENT.md`
- `.claude/settings.json`（`.claude` フォルダごと）
- `.gitignore.template`
- `security_audit_prompt.md`

### Step 2: VSCode でそのフォルダを開き、Claude Code を起動して下記プロンプトをコピペ

下の【プロンプト】の枠内を全選択してコピーし、Claude Code に貼り付けて送信してください。それだけです。あとは Claude Code が質問してくるので、答えながら進めます。

---

## 【プロンプト】（ここから下を全部コピーして Claude Code に貼り付け）

```
あなたはこのプロジェクトに対し、ナイスネイルグループのセキュリティ・開発ルールテンプレートを適用する作業を行います。私（このユーザー）は非エンジニアまたはセキュリティ専門家ではない可能性があります。技術用語を避け、業務上の意味で説明してください。

【絶対遵守事項】
1. 既存ファイルを勝手に上書きしない。既存ファイルがある場合は必ず差分を提示し、私の承認を得る
2. 指示された範囲外の変更を行わない
3. 推測で進めない。不明点は必ず質問する
4. Phase ごとに必ず一旦止まり、私の承認を得てから次に進む
5. 「問題ありません」「適切です」と楽観評価しない。発見した問題は最低5件報告する
6. シークレット値（パスワード・APIキー・トークン）の中身を絶対にチャットに出力しない（存在確認のみ）

==========================================
Phase 1: プロジェクト現状調査（読み取りのみ・変更なし）
==========================================

以下を調査して報告してください。技術用語には簡単な説明を付けてください。

A. プロジェクトの種類判定
   - これは何のアプリ・サイトか（package.json、index.html、README 等から推測）
   - 使用している主要な技術（フレームワーク、ライブラリ）
   - **デプロイ先（Netlify / ローカルのみ / その他）を必ず判定する**
     - netlify.toml の有無
     - .netlify/ ディレクトリの有無
     - package.json に netlify 関連の記述があるか

B. 既存ファイルの有無
   - CLAUDE.md は既にあるか
   - CLAUDE_security_baseline.md は既にあるか
   - .claude/settings.json は既にあるか
   - .gitignore は既にあるか（あれば内容も要約）

C. シークレット漏洩リスク調査（最重要）
   - .env ファイルの存在（中身は読まないこと）
   - .env が .gitignore に含まれているか
   - フロントエンドコードにハードコードされたパスワード・APIキー・トークンがないか
     （ソースファイルを grep で検索。発見しても値はチャットに出力しない）
   - git 履歴に過去シークレットがコミットされた可能性
   - リポジトリは Public か Private か（git remote -v で確認できる範囲で）

D. 認証・認可の現状
   - ログイン機能はあるか
   - 認証ロジックがどこに実装されているか
   - パスワードがコード内に平文で書かれていないか

E. データの取り扱い
   - 個人情報・機密情報を扱うアプリか
   - データの保存先はどこか（ローカルファイル、データベース、外部サービス等）

F. 即時危険の有無
   以下に該当する場合、最優先で警告してください：
   - パスワード・APIキーがコード内に平文で書かれている
   - パスワード・APIキーが git にコミットされている
   - リポジトリが Public で機密情報が含まれている
   - 認証なしで個人情報が取得できる API がある

報告は箇条書きで簡潔に。Phase 1 完了後、必ず一旦止まり、
「Phase 1 完了。Phase 2 に進んでよいですか？」と確認してください。

==========================================
Phase 2: テンプレート配置計画＆質問リスト
==========================================

Phase 1 を踏まえ、以下を提示してください：

A. 配置するファイル一覧
   各ファイルについて、新規配置 / 既存に追記 / 既存維持 のどれかを明記：
   - CLAUDE.md
   - CLAUDE_security_baseline.md
   - NETLIFY_DEPLOYMENT.md ← 【重要】Phase 1 で Netlify 使用と判定された場合のみ配置。ローカル開発のみなら配置不要と判断する
   - .claude/settings.json
   - .gitignore（追記内容を提示）
   - security_audit_prompt.md

B. ユーザーへの質問
   私（Claude Code）が判断できない情報を質問してください：
   - このプロジェクトの目的（業務上の意味）
   - 主な利用者
   - 取り扱う最も機密度の高いデータ
   - 認証方式（あれば）
   - データの保存先
   - Netlify を使う場合：サイト名、Site ID、公開URL、ビルドコマンド、公開ディレクトリ
     （Site ID は Netlify ダッシュボードの Site configuration → General → API ID で確認可能）

C. セキュリティリスク一覧
   発見したリスクを深刻度順に列挙：
   - 致命的（即時対応）：例「APIキーがソースコードに平文で記載され Public リポジトリで公開」
   - 高（1週間以内）
   - 中（1ヶ月以内）
   - 低
   各リスクについて、技術用語を使わず「何が起こりうるか」を業務影響で説明してください。
   例：「悪意ある人にこの API キーを使われ、月10万円規模の不正利用が発生する可能性」

Phase 2 完了後、必ず一旦止まり、私の回答を待ってください。

==========================================
Phase 3: テンプレート配置実行（私の承認後）
==========================================

私からの回答を受け取った後、以下を実行：

A. ファイル配置
   - 既存ファイルを上書きする場合、上書き直前にバックアップを取る
     （例：CLAUDE.md → CLAUDE.md.bak.YYYYMMDD）
   - Phase 1 で Netlify 不使用と判定した場合、NETLIFY_DEPLOYMENT.md は配置しない

B. CLAUDE.md の埋め込み
   - 私から得た情報を CLAUDE.md の【記入】箇所に反映
   - 不明な箇所は【記入】のまま残す

C. .claude/settings.json の調整
   - このプロジェクト固有の機密ファイルがあれば deny に追加（私の承認後）

D. .gitignore の更新
   - .gitignore.template の内容を既存 .gitignore にマージ（重複は除外、既存項目は削除しない）

E. 検証
   - 全ファイルが正しい場所にあるか
   - settings.json の JSON 構文チェック

完了後、サマリーを報告：
- 配置・追記・スキップしたファイル一覧
- 私が後で埋めるべき【記入】箇所のリスト
- 未対応のセキュリティリスクと推奨対応

「Phase 3 完了。git にコミットしてよいですか？」と確認してください。

==========================================
Phase 4: コミット
==========================================

A. git status を表示し、変更内容を確認

B. コミットメッセージ案を提示：
   「セキュリティ・開発ルールテンプレート適用」

C. push は私の判断に委ねる（自動でやらない）

==========================================
Phase 5: 動作確認手順の提示
==========================================

1. Claude Code を再起動し、/permissions でルールが反映されているか確認
2. cat .env を実行して deny されることを確認
3. CLAUDE.md の未記入箇所のリスト
4. 残っているセキュリティリスクと対応優先順位

==========================================
重要：シークレット保護
==========================================

- .env の中身を絶対に表示しない
- 発見したパスワード・トークン値をチャット出力に含めない
- 「このファイルにこういう種類のシークレットが含まれていそうです」程度の言及にとどめる

それでは Phase 1 から開始してください。
```

## 困った時のヒント

### Claude Code が一気に進めようとする
「Phase ごとに止まる指示です。中断してください」と返信。

### 「問題ありません」と返ってくる
「最低5件のリスクを発見してください。なければ改善余地でも構いません」と返信。

### 既存ファイルが上書きされた
ターミナルで以下を実行：
```
git checkout -- 上書きされたファイル名
```
コミット前なら復元できます。

### Site ID がわからない
1. https://app.netlify.com にログイン
2. 対象サイトをクリック
3. Site configuration → General → API ID
ここに表示される文字列が Site ID です。

### Netlify を使っていない
プロンプトの中で Claude Code が自動判定し、不要なファイルは配置しません。何も気にしなくて大丈夫です。

## このセットアップで得られるもの

- プロジェクトに潜むセキュリティリスクの可視化
- 今後 Claude Code に作業を依頼する際の事故防止ルール
- 個人情報・機密情報の漏洩リスク低減
- 誤ったデプロイ・新規サイトの誤作成防止（Netlify使用時）
